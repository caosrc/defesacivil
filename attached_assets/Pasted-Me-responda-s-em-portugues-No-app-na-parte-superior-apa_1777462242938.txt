USE ESSAS SOLUÇÕES: combinar 3 estratégias
1. 🔄 Atualização em tempo real (Realtime)

O Supabase já tem isso nativo.

Em vez de ficar dando fetch, você deve assinar mudanças:

supabase
  .channel('locations')
  .on(
    'postgres_changes',
    { event: '*', schema: 'public', table: 'locations' },
    (payload) => {
      console.log('Mudança:', payload)
      atualizarMapa(payload)
    }
  )
  .subscribe()

👉 Isso elimina boa parte do delay, porque o outro usuário recebe instantaneamente.

2. ⏱️ Heartbeat (checagem periódica inteligente)

Aqui está o pulo do gato que resolve o problema do GPS “fantasma”.

Cada usuário deve enviar atualizações frequentes tipo:

setInterval(() => {
  if (gpsAtivo) {
    enviarLocalizacao({
      lat,
      lng,
      last_seen: new Date(),
      is_active: true
    })
  }
}, 5000) // a cada 5s

No banco, você salva:

lat
lng
last_seen
is_active
3. 🧠 Regra de expiração (anti-delay / anti-fantasma)

Na hora de mostrar no mapa:

const AGORA = new Date()
const LIMITE = 10000 // 10 segundos

usuarios.forEach(user => {
  const ultimo = new Date(user.last_seen)
  
  const ativo = (AGORA - ultimo) < LIMITE && user.is_active

  if (ativo) {
    mostrarNoMapa(user)
  } else {
    removerDoMapa(user)
  }
})

👉 Isso resolve:

GPS desligado mas ainda visível
Conexão perdida
Delay de atualização
